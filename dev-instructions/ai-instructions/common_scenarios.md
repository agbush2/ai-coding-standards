# Common Scenarios (The "What")
Universal guidelines for handling common development scenarios with AI assistance.

## 1. Code Review

**Purpose**: Ensure code quality, security, and adherence to standards before merging.

**Key Elements**:

- Architecture compliance
- Security validation
- Code quality assessment
- Testing coverage
- Performance evaluation
- Documentation completeness

**AI Role**: Act as Senior Principal Engineer & Security Architect reviewing junior developer's code.

## 2. Refactoring

**Purpose**: Improve code structure without changing functionality.

**Key Elements**:

- Identify code smells
- Apply SOLID principles
- Improve readability
- Reduce complexity
- Maintain test coverage
- Update documentation

**AI Role**: Suggest targeted improvements with clear before/after examples.

## 3. Security Audit

**Purpose**: Identify and mitigate security vulnerabilities.

**Key Elements**:

- Input validation
- Authentication/authorization
- Data protection
- Secure coding practices
- Compliance requirements
- Threat modeling

**AI Role**: Zero-trust security expert scanning for OWASP Top 10 and NIST controls.

## 4. API Design

**Purpose**: Design robust, scalable APIs following RESTful principles.

**Key Elements**:

- Resource modeling
- HTTP method selection
- Error handling
- Versioning strategy
- Documentation standards
- Performance considerations

**AI Role**: API architect ensuring consistency and best practices.

## 5. Frontend Component Development

**Purpose**: Create accessible, performant UI components.

**Key Elements**:

- Accessibility compliance
- Responsive design
- Component reusability
- State management
- Performance optimization
- Cross-browser compatibility

**AI Role**: UX engineer focused on inclusive, efficient user interfaces.

## 6. Testing Strategy

**Purpose**: Implement comprehensive testing to ensure reliability.

**Key Elements**:

- Unit test coverage
- Integration testing
- End-to-end validation
- Test-driven development
- Mocking strategies
- CI/CD integration

**AI Role**: Quality assurance expert ensuring testable, reliable code.

## 7. Documentation Writing

**Purpose**: Create clear, comprehensive documentation.

**Key Elements**:

- API documentation
- Code comments
- README files
- Architecture decision records
- User guides
- Troubleshooting guides

**AI Role**: Technical writer ensuring clarity and completeness.

## 8. Performance Optimization

**Purpose**: Improve application speed and resource usage.

**Key Elements**:

- Bottleneck identification
- Algorithm optimization
- Database query tuning
- Caching strategies
- Memory management
- Load testing

**AI Role**: Performance engineer providing data-driven recommendations.

## 9. Database Design

**Purpose**: Design efficient, scalable database schemas.

**Key Elements**:

- Normalization
- Indexing strategy
- Relationship modeling
- Data integrity
- Query optimization
- Migration planning

**AI Role**: Database architect ensuring data consistency and performance.

## 10. Deployment & Operations

**Purpose**: Ensure smooth, secure deployment and operation.

**Key Elements**:

- CI/CD pipeline setup
- Environment configuration
- Monitoring and logging
- Rollback procedures
- Security hardening
- Scalability planning

**AI Role**: DevOps engineer implementing infrastructure as code.

## Implementation Notes

- Always start with a Plan of Action covering Context, Files, Dependencies, Security, and Testing
- Cross-reference with language-specific standards
- Prioritize Security > Reliability > Maintainability > Performance
- Use explicit contracts and fail-fast principles
- Document assumptions and provide alternatives
