---
**Skill Name**: Acceptance Criteria (Gherkin)  
**Version**: 1.0 (2026-02-02)  
**Persona(s)**: Product Manager, Quality Engineer, Developer  
**Description**:  
Translate requirements into testable Given/When/Then scenarios covering happy path, negative cases, and key edge cases.

**Usage Example**:
```prompt
Write acceptance criteria in Gherkin.
Include:
- At least 2 happy-path scenarios
- At least 2 negative/authorization scenarios
- Boundary and validation scenarios
Keep each scenario specific and verifiable.
```

**Implementation Notes**:
- [ ] Use domain language, not implementation details.
- [ ] Ensure every scenario has an observable outcome.
- [ ] Add a glossary if terms are ambiguous.
- [ ] Related to: requirements_gathering, test_case_generation
